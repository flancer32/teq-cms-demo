КУС — Когнитивно-Контекстное Управление Сайтом

КУС означает «Когнитивно-Контекстное Управление Сайтом». Для себя я использую формулу: «Создание контекста, управляющего Агентом, создающим контент». В английских материалах этот подход известен как AISM.

  • Я фиксирую цели и структуру в контексте.
  • Агент читает инструкции и обновляет файлы.
  • Контент публикуется без ручной синхронизации.

Архитектура продукта

Система состоит из TeqCMS, агента и когнитивного контекста. Контекст можно хранить отдельно от сайта.

КомпонентРоль
.gen.mdОписание задачи для страницы
.prompt.mdУточнение инструкций к генерации
agent/notes/...История запросов и результатов

Рабочий процесс

  1. Подготавливаю контекст.
  2. Ставлю задачу агенту.
  3. Агент обновляет шаблоны.
  4. Я принимаю Pull Request.

При необходимости я могу уточнить задачу и повторить итерацию.

Преимущества подхода

ПреимуществоОбъяснение
Git-контрольКаждая правка видна через git diff.
SSRСтраницы рендерятся на сервере по шаблонам.
Agent-reviewИзменения проходят ревью в Pull Request.
Массовое обновлениеПравки в контексте синхронно применяются к разным страницам.

Для кого это

Эта методика создана для индие-разработчиков и соло-фаундеров, которым нужны частые правки, автоматизация и отсутствие громоздкой CMS.
