AISM — managing a site through context and an agent

AISM stands for AI-Driven Site Management. I describe it as "Creating a context that guides the Agent who creates the content."

  • I outline goals and structure in the context.
  • The agent reads the instructions and edits the files.
  • The content is published without manual sync.

Product architecture

The system combines TeqCMS, the agent and the cognitive context. The context may live in a separate repository.

ComponentRole
.gen.mdPage generation task
.prompt.mdDetailed instructions
agent/notes/...Agent reports and history

Workflow

  1. Prepare the context.
  2. Define a task.
  3. The agent updates templates.
  4. I merge the Pull Request.

The task can be refined, and I can request another iteration before merging.

Approach advantages

AdvantageExplanation
Git controlEvery change is visible through git diff.
SSRTemplates render pages on the server.
Agent reviewChanges go through a Pull Request.
Mass updateContext edits cascade to many pages.

Who is it for

The approach targets indie developers and solo founders who need frequent edits, automation and no complex CMS.
