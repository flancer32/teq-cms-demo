Agent Driven Site Management (ADSM)

What Is It

ADSM is a way to manage a site through text instructions. You describe what pages you need and what should be on them, and the agent builds and maintains them.

The cognitive context is a set of files describing the project and the site structure. It helps the agent understand how the site is organized and which rules to follow.

Scheme


A human describes the task, the agent updates the site

Why It Matters

The site brings together social networks, videos and ads, so it must be updated regularly to stay useful.

Manual editing takes time. Developers, landing page authors and documentation writers face this constantly. ADSM removes the routine and lets you update content quickly.

How It Works

You write a text instruction: what the project is, which pages you need and what to place on them. The agent reads this text and creates or changes the site's files. You describe the goal — the agent does the work.

Example

Suppose you need an “About Us” page. You write: “Add an 'About Us' page with a title and a short team description.” The agent creates the page with the requested content. All interaction happens through plain text.
